⏺ 三级加载协议  
   
  三级加载协议是 vision-maker 项目中控制智能体加载文档粒度的核心规范，目的是避免上下文窗口被撑爆。它从两个维度运作：  
                                                          
  ---                                                                                                                                                  
  维度一：文档内部的三级加载                              

  ┌────────┬──────┬──────────────────────────────────────────┬───────────────┬───────────────────────────────────────────┐
  │  层级  │ 名称 │                 加载什么                 │     成本      │                   用途                    │
  ├────────┼──────┼──────────────────────────────────────────┼───────────────┼───────────────────────────────────────────┤
  │ Tier 1 │ 发现 │ front-matter 中的 description + concepts │ ~1-2 行       │ 快速判断：这篇文档跟当前任务有关吗？      │
  ├────────┼──────┼──────────────────────────────────────────┼───────────────┼───────────────────────────────────────────┤
  │ Tier 2 │ 加载 │ 文档正文                                 │ < 5000 tokens │ 获取核心知识                              │
  ├────────┼──────┼──────────────────────────────────────────┼───────────────┼───────────────────────────────────────────┤
  │ Tier 3 │ 深入 │ 正文引用的附属资源（图表、schema 等）    │ 按需          │ 正文不够详细时才去 references/ 目录找补充 │
  └────────┴──────┴──────────────────────────────────────────┴───────────────┴───────────────────────────────────────────┘

  直觉类比：就像读书 — 先看目录（Tier 1）→ 决定读哪章（Tier 2）→ 需要细节再翻附录（Tier 3）。 

  ---  
  维度二：SKILL.md 内部的三级加载  

  ┌────────┬───────────────────────────────────────────────────────────────┐
  │  层级  │                             内容                              │
  ├────────┼───────────────────────────────────────────────────────────────┤
  │ Tier 1 │ 模式概览 + 触发关键词（SKILL.md 的 description）              │
  ├────────┼───────────────────────────────────────────────────────────────┤
  │ Tier 2 │ 模式路由 + 响应协议摘要 + 自检流程（SKILL.md 正文，< 500 行） │
  ├────────┼───────────────────────────────────────────────────────────────┤
  │ Tier 3 │ 模式详细流程、方法论、任务规格（references/ 或项目文件）      │
  └────────┴───────────────────────────────────────────────────────────────┘

  ---  
  运作流程  

  1. 扫描 front-matter（Tier 1）：遍历所有文档，只读 description 和 concepts 两个字段，成本极低  
  2. 按需加载正文（Tier 2）：把判定为相关的文档正文拉进来，每篇控制在 5000 tokens 以内  
  3. 按需深入（Tier 3）：只有正文信息不够时，才去加载附属资源  

  为什么需要它  

  核心矛盾是：知识库可能很大，但上下文窗口是有限的。三级加载协议确保智能体不会"一股脑全加载"，而是像搜索引擎一样 — 先索引、再摘要、最后才深入细节。
